Initialization

There are two ways how init TOTI in your app: quick init and custom init. Quick init makes lot of work for you. On the other hand, custom init is more flexible.

Quick init

Quick init loads configuration from file, automatically create Database (see here) and Env (see here) objects.

If configuration for Database is not specified, Database instance will be NULL.

For quick init, you need instance of Application class. Constructor requires list of Module (see in next chapter). 


Application application = new Application(Arrays.asList(
	// modules
));

Configuration of Application is in .properties file. The file can be specified Application.APP_CONFIG_FILE = .... It must be called before constructor. Default is conf/app.properties. Can be in or outside classpath. The list of configurations is here. If you wish not use configuration file and let everything default, set Application.APP_CONFIG_FILE = null;

Control

start method does:

  1. Create and migrate database (if database and migrations used). See about migrations
  2. Run all tasks from modules (see in next chapter)
  3. Start HTTP server
If start fails, no exception is throwed but System.exit called.

stop method does:

  1. Stop HTTP server
  2. Stop all tasks from modules
If stop fails. no exception is throwed but System.exit called.

Application not throws exceptions. All action including exceptions are logged.

Custom init

For custom init, let start with HttpServerFactory. This class holds configuration for HttpServer. The list of configurations is here.

For creating HttpServe there are required two classes: Database (see here) and Env (see here). Both can be null. 


Database database = ...; // can be null
Env env = ...; // can be null
HttpServer server = new HttpServerFactory()
	// calling settings
	// ...
	.get(modules, env, database);

start method does:

  1. Run all tasks from modules (see in next chapter)
  2. Start HTTP server

stop method does:

  1. Stop HTTP server
  2. Stop all tasks from modules
