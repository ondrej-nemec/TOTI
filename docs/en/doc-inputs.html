Form inputs and Grid filters

Samples of forms and inputs are in Examples in part form

Samples of grids and filters are in Examples in part grid

Filters and inputs (controls) represent HTML inputs. Some parameters are predefined. If you need add another parameter, call addParam method (you can call moretimes).

Standart HTML5 form ignores disabled inputs. TOTI allows submit disabled inputs and ignore active ones. Most of TOTI controls contains parameters: disabled (setDisabled), exclude (setExclude, default false) a editable (setEditable, default value is same as in form).

How it works:

  • disabled == true: if exclude not specified, exclude is TRUE and input will not be submitted
  • disabled == true and exclude == true: input will not be submirted
  • disabled == true and exclude == false: input will be submitted
  • exclude == false and if disabled not specified: input will be submitted
  • if form is not editable and editable == false or not specified: form cannot be submitted, input will appears as plain text. Buttons and submits not appear.
  • if form is not editable and editable == true: form cannot be submitted and input displays as input

Actions

Submit

Submit button

Create input: Submit.create(String title, String name)

Optional parameters

  • Asynchronious submit setAsync - default value true
  • Set title: setTitle
  • Submit confirmation: setConfirmation: request for submit confirmation
  • Disable input: setDisabled
  • Submit policy: setSubmitPolicy: allowed values EXCLUDE (neodesílat), INCLUDE (odeslat), INCLUDE_ON_CLICK (submit Submit only if user click on it)
  • Value setValue: this value is send if submit is in submittingg data. Not Submit text
Only asynchronious submit
  • Redirect on success submit setRedirect: URL path. If submit returns JSON, TOTI iterates parameters and and in URL replace {--param-name--} given value.
  • Execute after success submit setOnSuccess: name of JS function. Input parameters are: submit response, Submit and form.
  • Execute after submit failure setOnFailure. name of JS function. Input parameters are: error, Submit, form. Not execute if response code is 400 - it is reserved for validation messages.

Response code 400

Response code 400 is reserved for validation message in response.

Button

Button in form that not submit data or button in grid with action. In grid, if URL contains '{*name-of-parameter*}', this part is replaced with correct value from row data. For example: '/{name}/edit/{id}' and row id is 2, so URL will be '/cars/edit/2'

Create: Button.create(String url, String name)

Optional parameters

  • Title setTitle
  • Asynchronious request setAsync: make async request or redirect to URL. Default value is true
  • Icon setIcon: TOTI add to title <i> with given class. Usuall used with FontAwesome or Bootstrap cons
  • Tooltip setTooltip
  • Add class addClass: can be called moretimes
  • Print condition (grids only): setCondition(jsCode, evaluate) during printing, TOTI executed given JS code. If result is true, button will appears. If second parameter is defined and is true, TOTI replace {*name-of-row-property*} with given row value
  • Submit confirmation setConfirmation: request for confirming submit or grid action. Text can contains parameters '{*name-of-parameter*}' (grid only).
Asynchronious request only
  • Method setMethod: GET, POST,...
  • Execute on action successsetOnSuccess: name of JS function. Input parameter is request result.
  • Execute on failure setOnFailure: name of JS function. Input parameter is error.
  • Add URL parameter addRequestParam: can be called moretimes

Load data from URL

Loading data from URL required following response format: 


[
	{
		value: '-value-',
		title: '-title-',
		optgroup: '-opt-group-', // optional
		disabled: true/false // optional
	},
	...
]

This result can be created using List of Help.

Depends

If Select ('Target') contains optgroups, it can depends on another Select ('Source'). If source has no selected value, target display normally (but optgroups in target are replaced with source options title: target.optgroup = sourceOption.id => target display sourceOption.title as optgroup) If source contais value, target display only options without optgroup or with optgroup same as source value.

InputList is container for another inputs. These inputs are available on page with name InputListName[inputName] (in TOTI called Map) or InputListName[] (in TOTI called List). Input only

Create input: InputList.input(String name). If you wish use list let name empty string (not null).
Add input: inputList.addInput(Input input)
