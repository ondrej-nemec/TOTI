Formulářové vstupy a filtry tabulek

Ukázky použití formuláře a vstupů je v Examples v části form

Ukázky použití tabulek a filtrů je v Examples v části grid

Filtry a vstupy (controls) reprezentují HTML inputs. Mají některé předpřipravené parametry. Pokud potřebujete použít jiný, zavolejte metodu addParam (je možné ji volat vícekrát zasebou).

Standartní HTML5 formulář ignoruje neaktivní vstupy (disabled inputs). TOTI naopak umožňuje odeslat neaktivní vstupy a naopak ignorovat aktivní. Většina TOTI controls obsahuje parametry: disabled (setDisabled), exclude (setExclude, default false) a editable (setEditable, výchozí hodnota je stejná s formulářem).

Jak to funguje:

  • disabled == true: pokud exclude není nastaveno, exclude je TRUE a vstup nebude odeslán
  • disabled == true a zároveň exclude == true: vstup nebude odeslán
  • disabled == true a zároveň exclude == false: vstup bude odeslán
  • exclude == false a disabled není nastaven: vstup bude odeslán
  • pokud formulář není editovatelný a editable == false nebo není nastaven: formulář nelze odeslat, vstup se zobrazí jako obyčejný text (Tlačítko na odeslání formuláře se nezobrazí).
  • pokud formulář není editovatelný a editable == true: formulář nelze odeslat a vstup se zobrazí normálně

Akce

Submit

Odesílací tlačítko

Vytvoření vstupu: Submit.create(String title, String name)

Volitelné parametry

  • Asynchronní odeslání setAsync - výchozí hodnota je true
  • Nastavení popisu: setTitle
  • Potvrzení odeslání: setConfirmation: žádost o potvrzení odeslání formuláře
  • Neaktivní vstup: setDisabled
  • Odesílat Submit: setSubmitPolicy: povolené hodnoty EXCLUDE (neodesílat), INCLUDE (odeslat), INCLUDE_ON_CLICK (odeslat jen pokud na něj bylo kliknuto)
  • Hodnota setValue: hodnota, která se pošle, pokud se Submit odesílá. Nejedná se text tlačítka
Jen při asychnroním odeslání
  • Přesměrování po úspěšném odeslání setRedirect: URL cesta. Pokud odeslání vrátilo JSON, TOTI projde všechny jeho parametry a v URL nahradí {--jmeno-parametru} danou hodnotou.
  • Provést po úspěšném odeslání setOnSuccess: název JS funkce. Vstupní parametry jsou: výsledek odeslání, Submit a formulář.
  • Provést při chybě setOnFailure: název JS funkce. Vstupní parametry jsou: errror, Submit, formulář. Neprovede se při chybě 400 - ta je rezervovaná pro validační hlášky

Chyba 400

Chya 400 je rezervovaná pro situace, kdy data ve formuláři nejsou validní. Tato odpověď pak obsahuje detail chyb.

Button

Tlačtko ve formuláři, které neodešle data, nebo tlačítko v tabulce, které provede akci. Pokud je Button přidán jako tlačítko k řádku tabulky a URL obsahuje '{*name-of-parameter*}', je tato část nahrazena odpovídající hodnotou z dat řádku. Například: '/{name}/edit/{id}' a id řádku je 2, pak URL bude '/cars/edit/2'

Vytvoření: Button.create(String url, String name)

Volitelné parametry

  • Popisek setTitle
  • Asynhronní odeslání setAsync: určuje, zda provést asynchronní akci, nebo přesměrovat. Výchozí je true
  • Ikona setIcon: TOTI přídá k popisku <i> s dannou třídou. Vhodné při použití FontAwesome nebo Bootstrap icons
  • Nápověda setTooltip
  • Přidat třídu addClass: lze volat vícekrát
  • Vykreslovací podmínka (jen u tabulek): setCondition(jsCode, evaluate) - při vykresolování, TOTI provede zadaný JS kód. Pokud je výsledek true, tlačítko se zobrazí. Druhý parametr určuje, zda JS kód obsahuje zástupné znaky pro data řádku {*name-of-row-property*}
  • Potvrzení odeslání setConfirmation: žádost o potvrzení provedení akce. Text může obsahovat zástupné znaky pro data řádku '{*name-of-parameter*}'.
Jen při asynchroním odeslání
  • Metoda setMethod: GET, POST,...
  • Provést při úspěšném odeslání setOnSuccess: název JS funkce. Vstupním parametrem je výsledek požadavku.
  • Provést při chybě odeslání setOnFailure: název JS funkce. Vstupním parametrem je výsledek error.
  • Přidání URL parametru addRequestParam: může být voláno vícekrát

Načtení dat z URL

Pro načtení možností do selectu je vyžadován následujíc formát dat: 


[
	{
		value: '-value-',
		title: '-title-',
		optgroup: '-opt-group-', // optional
		disabled: true/false // optional
	},
	...
]

Tento výsledek poskytne List of Help.

Depends

Pokud Select ('Cíl') obsahuje optgroupy, může záviset na jiném Select ('Zdroj'). Možnosti Zdroje odpovídají skupinám možností v Cíli. Podud Zdroj nemá vybranou žádnou hodnotu, možnosti v Cíli se zobrazí rozdělené do skupin, přičemž názvy skupin jsou nahrazeny popisy možností ze Zdroje. Pokud je ve Zdroji vybraná možnost, v cíli se zobrazí možnosti s odpovídající skupinou a ty bez skupiny.

Speciálně v TOTI

InputList

InputList je kontejner pro jiné vstupy. Tyto vstupy jaou pak na stránce pojmenované InputListName[inputName] (mapa) nebo InputListName[] (list). Jen pro vstupy

Vytvoření vstupu: InputList.input(String name). Pokud chcete použít list, nechte jméno prázdné (ne null).
Přidání vstupu: inputList.addInput(Input input)

Ukázka použití je v příkladech.

DynamicList

DynamicList je také kontejner na vstupy. Rozdíl oproti InputList je v tom, že DynamicList umožňuje přidat definované vstupy n-krát přímo na stránce. Uživatel ovládá DynamicList pomocí tlačítek Přidat a Odebrat. Pro jména vstupů platí stejná pravidla jako u InputListu. Pokud popis přidaného vstupu obsahuje {i}, je tato část nahrazena indexem v rámci DynamicList (číslováno od 1). Jen pro vstupy

Vytvoření vstupu: DynamicList.input(String name)
Přidání vstupu: dynamicList.addInput(Input input);

Volitelné parametry

  • Popis setTitle: label of input
  • Zobrazit tlačítko Přidat: useAddButton
  • Zobrazit tlačítko Odebrat: useRemoteButton
  • Přidat první nevyplněnou položku: setAddFirstBlank. Výchozí hodnota je true
  • Načtení vstupů na základě URL: 
    
setLoadDat(String url, String method);
// OR with parameters
setLoadData(String url, String method, Map<String, String> params);
    viz níže

Načtení vstupů podle URL

TOTI udělá požadavek na danou URL. Očekávaný formát je stejný jako u Selectu. Kolik je položek v načtených datech, tolikrát se provede přidání vstupů. Pokud jméno přidaného vstupu obsahuje {i}, je tato část nahrazena title parametrem z dat.

Ukázka použití je v příkladech.
